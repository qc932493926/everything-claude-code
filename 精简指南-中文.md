## 技能和命令

技能像规则一样运作，限制在某些范围和工作流程内。当你需要执行特定工作流程时，它们是提示的缩写。

经过与 Opus 4.5 的长期编码会话后，想清理死代码和松散的 .md 文件？运行 `/refactor-clean`。需要测试？运行 `/tdd`、`/e2e`、`/test-coverage`。技能还可以包含代码地图 - 一种让 Claude 快速导航代码库而不浪费上下文探索的方式。

![显示链接命令的终端](./assets/images/shortform/02-chaining-commands.jpeg)
*链接命令一起*

命令是通过斜杠命令执行的技能。它们重叠但存储方式不同：

- **技能**：`~/.claude/skills/` - 更广泛的工作流定义
- **命令**：`~/.claude/commands/` - 快速可执行的提示

```bash
# 示例技能结构
~/.claude/skills/
  pmx-guidelines.md      # 项目特定的模式
  coding-standards.md    # 语言最佳实践
  tdd-workflow/          # 包含 README.md 的多文件技能
  security-review/       # 基于清单的技能
```

## 钩子

钩子是在特定事件上触发的基于触发器的自动化。与技能不同，它们限制在工具调用和生命周期事件内。

**钩子类型：**

1. **PreToolUse** - 在工具执行前（验证、提醒）
2. **PostToolUse** - 工具完成后（格式化、反馈循环）
3. **UserPromptSubmit** - 当你发送消息时
4. **Stop** - 当 Claude 完成响应时
5. **PreCompact** - 上下文压缩前
6. **Notification** - 权限请求

**示例：长运行命令前的 tmux 提醒**

```json
{
  "PreToolUse": [
    {
      "matcher": "tool == \"Bash\" && tool_input.command matches \"(npm|pnpm|yarn|cargo|pytest)\"",
      "hooks": [
        {
          "type": "command",
          "command": "if [ -z \"$TMUX\" ]; then echo '[Hook] 考虑为会话持久性使用 tmux' >&2; fi"
        }
      ]
    }
  ]
}
```

![PostToolUse 钩子反馈](./assets/images/shortform/03-posttooluse-hook.png)
*在 Claude Code 中运行 PostToolUse 钩子时收到的反馈示例*

**专业提示：** 使用 `hookify` 插件来对话地创建钩子，而不是手动编写 JSON。运行 `/hookify` 并描述你想要的内容。

## 子代理

子代理是你的编排器（主要 Claude）可以委托任务给具有有限范围的进程。它们可以在后台或前台运行，为主代理释放上下文。

子代理与技能配合得很好 - 能够执行技能子集的子代理可以被委派任务并自主使用这些技能。它们也可以用特定工具权限进行沙箱化。

```bash
# 示例子代理结构
~/.claude/agents/
  planner.md           # 功能实现规划
  architect.md         # 系统设计决策
  tdd-guide.md         # 测试驱动开发
  code-reviewer.md     # 质量/安全审查
  security-reviewer.md # 漏洞分析
  build-error-resolver.md
  e2e-runner.md
  refactor-cleaner.md
```

为每个子代理配置允许的工具、MCP 和权限以实现正确的范围。

## 规则和记忆

你的 `.rules` 文件夹包含 Claude 应该始终遵循的带有最佳实践的 .md 文件。两种方法：

1. **单个 CLAUDE.md** - 一个文件中的所有内容（用户或项目级别）
2. **规则文件夹** - 按关注点分组的模块化 .md 文件

```bash
~/.claude/rules/
  security.md      # 无硬编码密钥，验证输入
  coding-style.md  # 不可变性、文件组织
  testing.md       # TDD 工作流、80% 覆盖率
  git-workflow.md  # 提交格式、PR 流程
  agents.md        # 何时委派给子代理
  performance.md   # 模型选择、上下文管理
```

**示例规则：**

- 代码库中没有表情符号
- 在前端中避免紫色色调
- 部署前始终测试代码
- 优先选择模块化代码而不是超大文件
- 永远不要提交 console.logs

## MCP（模型上下文协议）

MCP 直接将 Claude 连接到外部服务。不是 API 的替代品 - 它是围绕它们的提示驱动包装器，允许在导航信息时更灵活。

**示例：** Supabase MCP 让 Claude 提取特定数据，直接在上游运行 SQL 而无需复制粘贴。数据库、部署平台等也是如此。

![Supabase MCP 列出表](./assets/images/shortform/04-supabase-mcp.jpeg)
*Supabase MCP 列出公共模式中的表的示例*

**Claude 中的 Chrome：** 是一个内置的插件 MCP，让 Claude 自主控制你的浏览器 - 单击周围查看事情如何工作。

**关键：上下文窗口管理**

对 MCP 要有选择性。我将所有 MCP 保留在用户配置中，但**禁用所有未使用的**。导航到 `/plugins` 并向下滚动或运行 `/mcp`。

![/plugins 界面](./assets/images/shortform/05-plugins-interface.jpeg)
*/plugins 导航到 MCP 以查看当前安装的 MCP 和它们的状态*

你的 200k 上下文窗口在启用过多工具时可能只有 70k。性能会显著下降。

**经验法则：** 配置中有 20-30 个 MCP，但每个项目保持在 10 个以内启用 / 低于 80 个活跃工具。

```bash
# 检查启用的 MCP
/mcp

# 在 ~/.claude.json 下的 projects.disabledMcpServers 中禁用未使用的
```

## 插件

插件打包工具以便于安装，而不是繁琐的手动设置。插件可以是技能 + MCP 的组合，或者绑定在一起的钩子/工具。

**安装插件：**

```bash
# 添加市场
claude plugin marketplace add https://github.com/mixedbread-ai/mgrep

# 打开 Claude，运行 /plugins，找到新市场，从那里安装
```

![显示 mgrep 的市场选项卡](./assets/images/shortform/06-marketplaces-mgrep.jpeg)
*显示新安装的 Mixedbread-Grep 市场*

**LSP 插件**如果你经常在编辑器外运行 Claude Code 特别有用。语言服务器协议为 Claude 提供实时类型检查、转到定义和智能完成，而无需打开 IDE。

```bash
# 启用的插件示例
typescript-lsp@claude-plugins-official  # TypeScript 智能
pyright-lsp@claude-plugins-official     # Python 类型检查
hookify@claude-plugins-official         # 对话地创建钩子
mgrep@Mixedbread-Grep                   # 比 ripgrep 更好的搜索
```

与 MCP 相同的警告 - 注意你的上下文窗口。

## 提示和技巧

### 键盘快捷键

- `Ctrl+U` - 删除整行（比反复退格更快）
- `!` - 快速 bash 命令前缀
- `@` - 搜索文件
- `/` - 启动斜杠命令
- `Shift+Enter` - 多行输入
- `Tab` - 切换思考显示
- `Esc Esc` - 中断 Claude / 恢复代码

### 并行工作流

- **Fork** (`/fork`) - Fork 对话以在不重叠的任务中并行做，而不是发送排队的消息
- **Git Worktrees** - 对于重叠的并行 Claude 而不会冲突。每个 worktree 是独立的检出

```bash
git worktree add ../feature-branch feature-branch
# 现在在每个 worktree 中运行单独的 Claude 实例
```

### 长运行命令的 tmux

流式传输和观看 Claude 运行的日志/bash 进程：

https://github.com/user-attachments/assets/shortform/07-tmux-video.mp4

```bash
tmux new -s dev
# Claude 在这里运行命令，你可以分离并重新附加
tmux attach -t dev
```

### mgrep > grep

`mgrep` 是相对于 ripgrep/grep 的重大改进。通过插件市场安装，然后使用 `/mgrep` 技能。适用于本地搜索和网络搜索。

```bash
mgrep "function handleSubmit"  # 本地搜索
mgrep --web "Next.js 15 app router changes"  # 网络搜索
```

### 其他有用的命令

- `/rewind` - 回到以前的状态
- `/statusline` - 用分支、上下文 %、待办事项自定义
- `/checkpoints` - 文件级撤销点
- `/compact` - 手动触发上下文压缩

### GitHub Actions CI/CD

使用 GitHub Actions 在你的 PR 上设置代码审查。配置后，Claude 可以自动审查 PR。

![Claude bot 批准 PR](./assets/images/shortform/08-github-pr-review.jpeg)
*Claude 批准错误修复 PR*

### 沙箱化

对危险操作使用沙箱模式 - Claude 在受限环境中运行而不影响你的实际系统。

## 关于编辑器

你的编辑器选择显著影响 Claude Code 工作流程。虽然 Claude Code 可以从任何终端工作，将其与功能强大的编辑器配对会解锁实时文件跟踪、快速导航和集成命令执行。

### Zed（我的偏好）

我使用 [Zed](https://zed.dev) - 用 Rust 编写，所以它的速度很快。即时打开，毫不费力地处理大型代码库，几乎不接触系统资源。

**为什么 Zed + Claude Code 是很好的组合：**

- **速度** - 基于 Rust 的性能意味着在 Claude 快速编辑文件时没有延迟。你的编辑器跟上
- **代理面板集成** - Zed 的 Claude 集成让你在 Claude 编辑时实时跟踪文件更改。跳转 Claude 引用的文件而无需离开编辑器
- **CMD+Shift+R 命令面板** - 快速访问所有自定义斜杠命令、调试器、构建脚本在可搜索的 UI 中
- **最小资源使用** - 在繁重操作期间不会与 Claude 竞争 RAM/CPU。在运行 Opus 时很重要
- **Vim 模式** - 完整的 vim 键绑定如果那是你的风格

![带有自定义命令的 Zed 编辑器](./assets/images/shortform/09-zed-editor.jpeg)
*Zed 编辑器中使用 CMD+Shift+R 的自定义命令下拉菜单。跟随模式显示为右下角的靶心。*

**编辑器无关的提示：**

1. **分割你的屏幕** - 一侧是带有 Claude Code 的终端，另一侧是编辑器
2. **Ctrl + G** - 快速在 Zed 中打开 Claude 当前工作的文件
3. **自动保存** - 启用自动保存以便 Claude 的文件读取始终是当前的
4. **Git 集成** - 在提交前使用编辑器的 git 功能审查 Claude 的更改
5. **文件观察者** - 大多数编辑器自动重新加载更改的文件，验证这是启用的

### VSCode / Cursor

这也是一个可行的选择并与 Claude Code 配合得很好。你可以在终端格式中使用它，使用 `\ide` 启用自动同步与你的编辑器，启用 LSP 功能（现在有点冗余插件）。或者你可以选择与编辑器更集成的扩展，具有匹配的 UI。

![VS Code Claude Code 扩展](./assets/images/shortform/10-vscode-extension.jpeg)
*VS Code 扩展提供 Claude Code 的本机图形界面，直接集成到你的 IDE 中。*


## 关键要点

1. **不要过度复杂化** - 把配置看作微调，而不是架构
2. **上下文窗口很宝贵** - 禁用未使用的 MCP 和插件
3. **并行执行** - fork 对话，使用 git worktrees
4. **自动化重复的** - 钩子用于格式化、linting、提醒
5. **限制你的子代理范围** - 有限工具 = 专注执行
